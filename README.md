## Overview

PasswordNexus is a full-stack web application built with **Python + Flask** as an ICT course project at Air University, Islamabad. It provides a complete password security toolkit with five integrated modules, persistent SQLite logging, and a clean browser-based interface.

## Module Details

### рҹ”җ Password Generator
Uses Python's `secrets` module (cryptographically secure RNG) to generate passwords with guaranteed character class coverage:
- Enforces at least 1 lowercase, 1 uppercase, 1 digit, 1 symbol
- Minimum length: 8 В· Maximum: 100
- Character pool: `A-Z a-z 0-9 !@#$%&*()`
- Final password is shuffled with `random.shuffle()` to eliminate positional bias

### рҹ§Ә Strength Analyzer
Two-stage analysis pipeline:
1. **Policy check** вҖ” regex validation for uppercase, lowercase, digit presence + minimum length
2. **zxcvbn scoring** вҖ” realistic strength estimation accounting for dictionary words, patterns, and common substitutions. Returns score (0вҖ“4), crack time, and improvement suggestions.

### вҸұпёҸ Brute Force Estimator
Mathematical crack-time calculator:

```
combinations = charset_size ^ password_length
time = combinations / attempts_per_second
```

Charset options: lowercase only (26) В· mixed case (52) В· alphanumeric (62) В· full ASCII printable (95)

### рҹ”„ Hash Converter
Wraps Python's `hashlib` to convert any text to: MD5 В· SHA1 В· SHA224 В· SHA256 В· SHA384 В· SHA512 В· and any other algo in `hashlib.algorithms_available`

### рҹ“Ұ Base Encoder/Decoder
Supports: Base64 В· Base32 В· Base16 (hex) В· URL encoding/decoding вҖ” bidirectional, with terms agreement gate.

## Activity Logging

Every tool usage is recorded in `nexus.db`:

```sql
CREATE TABLE logs (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    type       TEXT NOT NULL,        -- e.g. "Generator", "Analyzer", "Hash"
    detail     TEXT NOT NULL,        -- e.g. "Generated pass length: 16"
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

On shutdown, logs are exported to `log.txt` automatically via `export_logs()`.

---

## Security Notes

- Passwords are generated using `secrets.choice()` вҖ” **not** `random.choice()`. The `secrets` module uses OS-level entropy (suitable for cryptographic use).
- No passwords are stored anywhere вҖ” only event metadata is logged.
- Hash conversion is one-way; the app never stores input text.
